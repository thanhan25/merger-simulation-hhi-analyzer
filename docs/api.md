
# API Reference

This document summarizes the main functions and commands provided by the `merger_sim` package. It is organized into library functions (Python) and userŌĆæfacing interfaces (CLI and Streamlit UI).

## Library Functions

All library functions live under `src/merger_sim`. The most important ones are in `metrics.py`, `simulation.py`, and `io.py`.

### `merger_sim.metrics`

#### `calculate_hhi(market_shares: pd.Series) -> float`

Compute the HerfindahlŌĆōHirschman Index from a series of market shares (in percent).

**Parameters**

- `market_shares`: `pd.Series` of numeric percentages.

**Validation**

- Raises `ValueError` if any share is negative.
- Raises `ValueError` if the sum of shares exceeds 100.1.

**Returns**

- `float` ŌĆō HHI value as the sum of squared shares.

#### `evaluate_antitrust_risk(hhi_pre: float, hhi_post: float) -> Tuple[str, str]`

Assess the antitrust risk of a merger using concentration categories and change in HHI.

**Parameters**

- `hhi_pre`: preŌĆæmerger HHI.
- `hhi_post`: postŌĆæmerger HHI.

**Returns**

- `(concentration_level, risk_label)`:

  - `concentration_level`: `"Unconcentrated"`, `"Moderately Concentrated"`, or `"Highly Concentrated"`.
  - `risk_label`: humanŌĆæreadable string summarizing risk (e.g. `"High Risk: Presumed to enhance market power (Red Flag)."`).

### `merger_sim.simulation`

#### `simulate_merger(df: pd.DataFrame, firm_col: str, share_col: str, acquirer: str, target: str) -> dict`

Simulate a merger between two firms and compute preŌĆæ and postŌĆæmerger HHI and risk metrics.

**Parameters**

- `df`: Data frame with at least `firm_col` and `share_col`.
- `firm_col`: name of the firm column (e.g. `"Firm"`).
- `share_col`: name of the share column (e.g. `"Market_Share"`).
- `acquirer`: name of the acquiring firm.
- `target`: name of the target firm.

**Returns**

A dictionary with keys:

- `hhi_pre` ŌĆō HHI before the merger.
- `hhi_post` ŌĆō HHI after the merger.
- `delta` ŌĆō difference `hhi_post - hhi_pre`.
- `concentration_level` ŌĆō qualitative category.
- `regulatory_risk` ŌĆō descriptive risk label.
- Optionally, additional fields used by plotting/UI components.

Raises `ValueError` if the specified firms are not found or if share data fails validation.

### `merger_sim.io`

#### `load_market_data(filepath: str) -> pd.DataFrame`

Load market data from a local CSV file.

**Behavior**

- Reads the file using `pandas.read_csv`.
- Trims whitespace from column names.
- Requires columns `Firm` and `Market_Share`.
- Converts `Market_Share` to numeric, dropping rows with nonŌĆænumeric values.

Raises:

- `FileNotFoundError` if the path does not exist.
- `ValueError` if required columns are missing.

#### `load_market_data_from_bigquery(query: str) -> pd.DataFrame`

Execute a SQL query against BigQuery and return standardized market data.

**Behavior**

- Uses `google.cloud.bigquery.Client` to run the query.
- Requires a `Firm` column.
- Accepts either:
  - `Market_Share` column (used directly), or
  - `Revenue` column (converted to shares by dividing by total revenue and multiplying by 100).
- Returns a data frame with columns `Firm` and `Market_Share`.

Raises:

- `RuntimeError` if BigQuery fails or authentication is unavailable.
- `ValueError` if required columns are missing.

## Command Line Interface (CLI)

The CLI is defined in `cli.py` and exposed as the `merger-sim` script via `project.scripts` in `pyproject.toml`.

Run commands with:

```bash
uv run merger-sim <command> [options]
```

### `merger-sim analyze`

Simulate a merger and print the results to the terminal, optionally generating plots.

**Signature**

```bash
uv run merger-sim analyze ACQUIRER TARGET [--file PATH] [--query SQL] [--mock]
```

**Options**

- `--file, -f PATH` ŌĆō path to a local CSV file with `Firm` and `Market_Share`.
- `--query, -q SQL` ŌĆō BigQuery SQL query string.
- `--mock` ŌĆō use a builtŌĆæin mock dataset instead of loading data.

At least one of `--file`, `--query`, or `--mock` is required.

**Output**

- A Rich panel with the merger description.
- A Rich table with pre/post HHI, delta, concentration level, and risk label.
- A plot generated by `plot_merger_impact(results)`.

Returns nonŌĆæzero exit codes for invalid inputs, data loading failures, or simulation errors.

### `merger-sim ui`

Launch the Streamlit dashboard.

```bash
uv run merger-sim ui
```

This command starts `streamlit run src/merger_sim/app.py` and prints a message indicating that the server is starting.

---

## Streamlit Dashboard

The dashboard in `app.py` mirrors the CLI functionality with a web UI:

- Upload CSV files.
- Configure acquirer/target firms.
- Trigger simulations with buttons.
- View HHI metrics and risk labels alongside interactive charts.

While the dashboard API is not intended as a programmatic interface, you can reuse helper functions from `app.py` in your own Streamlit apps.

## Stability and Versioning

The library API (`metrics`, `simulation`, `io`) is the most stable part of the project and will evolve conservatively. CLI commands and the UI may gain options over time, but core behavior is kept backwardsŌĆæcompatible where possible.

Breaking changes will be noted in the `CHANGELOG.md` and releases.
